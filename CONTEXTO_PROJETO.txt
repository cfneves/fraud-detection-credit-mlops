━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Pipeline de Machine Learning de nível produção para DETECÇÃO DE FRAUDE
em cartões de crédito, com:

  - Modelo LightGBM calibrado (isotonic regression)
  - Balanceamento SMOTE (sampling_ratio=0.1)
  - Métricas de negócio: KS Statistic, Gini, PSI, Brier Score
  - Explicabilidade SHAP (global + local + relatório HTML)
  - Dashboard Streamlit DIDÁTICO (para usuários leigos)
  - Deploy ativo no Streamlit Cloud

Dataset principal: Credit Card Fraud Detection (Kaggle mlg-ulb)
  - 284.807 transações reais anonimizadas (PCA → V1-V28)
  - 492 fraudes (0,17% — desbalanceamento extremo)
  - Arquivo: data/raw/fraud/creditcard.csv (66 MB, NÃO está no git)

6. COMO RODAR EM UMA MÁQUINA NOVA (PASSO A PASSO)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  PASSO 1 — Clonar o repositório
    git clone https://github.com/cfneves/fraud-detection-credit-mlops.git
    cd fraud-detection-credit-mlops

  PASSO 2 — Instalar dependências
    pip install -r requirements.txt
    (Python 3.10+ recomendado)

  PASSO 3 — Configurar Kaggle API (para dataset completo)
    - Acesse: https://www.kaggle.com/settings → Create New Token
    - Salve o kaggle.json em: ~/.kaggle/kaggle.json

  PASSO 4 — Baixar o dataset real
    python data_download.py --dataset fraud
    (baixa 66MB → salva em data/raw/fraud/creditcard.csv)

  PASSO 5 — Rodar o pipeline de ML
    python pipeline.py --dataset fraud --quick    ← rápido (LightGBM)
    python pipeline.py --dataset fraud --full     ← completo (todos + stacking)

  PASSO 6 — Abrir o dashboard
    streamlit run app.py
    (abre em http://localhost:8501)

  ATENÇÃO: sem o dataset real (passo 3-4), o app funciona com
  a amostra sintética de 2.000 linhas que já está no repositório.

7. DECISÕES TÉCNICAS IMPORTANTES (POR QUE FEZ ASSIM)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  RobustScaler (não StandardScaler)
    → Transações financeiras têm outliers extremos. RobustScaler usa mediana/IQR.

  PR-AUC como métrica principal (não ROC-AUC)
    → Com 0.17% de fraudes, ROC-AUC pode ser enganoso. PR-AUC foca nas fraudes.

  SMOTE com sampling_ratio=0.1 (não 0.5)
    → Oversamplear para 50/50 introduz muito ruído sintético. 10% é suficiente.

  CalibratedClassifierCV (isotonic regression)
    → LightGBM produz probabilidades extremas. Calibração corrige para compliance.

  Threshold ótimo por custo (FN=R$100, FP=R$5)
    → Fraude não detectada custa 20x mais que falso alarme. Threshold=0.1005.

  KS/Gini muito altos (89.7/89.8)
    → Porque V1-V28 são features PCA pré-otimizadas. Em produção real: KS 40-60.

  Amostra sintética no git (2.000 linhas)
    → Dataset real tem 66MB — acima do limite GitHub/Streamlit Cloud.
       Amostra garante que o demo online funcione sem credenciais do Kaggle.
       Métricas reais estão hardcoded em REAL_METRICS no app.py.

  src/ como pacote Python (com __init__.py)
    → Permite importar com "from src.metrics import ..." de qualquer lugar.
       Viabiliza futuramente "pip install -e ." para instalação local.
